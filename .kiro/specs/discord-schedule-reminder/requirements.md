# Requirements Document

## Project Description (Input)
現在、私が所属している部活で毎週活動があるのですが、毎回リマインドすると忘れることがあるのでこれを自動化したいです。今考えている実装は、Discordの特定のチャンネルから予定を読み取り、今日が活動日の週になったらBotでリマインドを送らせたいです。PythonとGitHub Actionsでできると考えています。

## Requirements

### 機能要件

#### FR-1: 予定読み取り機能
- **FR-1.1**: Discordの指定された予定チャンネルからメッセージを取得する
- **FR-1.2**: メッセージから日付と活動内容を抽出する（形式: `M/D 活動内容`）
- **FR-1.3**: 当週（水曜日）に該当する予定を特定する

#### FR-2: リマインド送信機能
- **FR-2.1**: 活動週の前週金曜日に1回目のリマインドを送信する
  - 目的: 土日に活動準備をしてもらうため
- **FR-2.2**: 活動週の月曜日に2回目のリマインドを送信する
  - 目的: 発表スライドなどの最終準備を促すため
- **FR-2.3**: リマインドは「重要連絡」チャンネルに送信する
- **FR-2.4**: リマインドメッセージには活動日と活動内容を含める
- **FR-2.5**: リマインドメッセージに@everyoneメンションを付けて全員に通知する

#### FR-3: 自動実行機能
- **FR-3.1**: GitHub Actionsで毎週金曜日と月曜日に定期実行する
- **FR-3.2**: 該当する予定がない場合はリマインドを送信しない

### 非機能要件

#### NFR-1: 技術スタック
- **NFR-1.1**: Python 3.xで実装する
- **NFR-1.2**: GitHub Actionsのスケジュール実行（cron）を使用する
- **NFR-1.3**: Discord Bot API（discord.pyまたはHTTP API）を使用する

#### NFR-2: 設定管理
- **NFR-2.1**: チャンネルID（予定チャンネル、リマインドチャンネル）は環境変数で管理する
- **NFR-2.2**: Discord Bot Tokenは GitHub Secrets で安全に管理する

#### NFR-3: 信頼性
- **NFR-3.1**: 予定形式のパースエラー時は処理をスキップしログを出力する
- **NFR-3.2**: API接続エラー時は適切なエラーメッセージを出力する

### 制約事項
- **C-1**: 活動日は基本的に毎週水曜日である
- **C-2**: 予定の投稿形式は `M/D 活動内容`（例: `3/20 練習試合`）とする
- **C-3**: リマインドタイミングは活動週の前週金曜日と当週月曜日の2回

### ユースケース

#### UC-1: 週次リマインドフロー
1. GitHub Actionsが金曜日または月曜日にワークフローを起動
2. Botが予定チャンネルから直近のメッセージを取得
3. 次の水曜日に該当する予定があるか確認
4. 該当予定があれば、重要連絡チャンネルにリマインドを送信
5. 該当予定がなければ、何もせず終了

#### UC-2: リマインドメッセージ例
```
@everyone
📅 今週の活動予定
日付: 3/20（水）
内容: 練習試合

準備をお願いします！
```
